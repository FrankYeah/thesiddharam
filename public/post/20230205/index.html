收過 40、50 封履歷後的心得

舉三個應徵情境，你會選誰合作？


文 / 西打藍 Siddharam

這幾天，自己做了一件重大決定：結束合作兩年多的網站開發合夥關係。未來再發一篇文細聊這件事，加上自己案源漸增，於是在多個社群上，徵求有意願接案的前端工程師。

目前約收到近十封履歷與作品集，加上過往接案與經營社群過程，很常收到不同領域的履歷，包含：平面設計師、UI/UX 設計師、後端工程師、文案、剪輯師、插畫、翻譯、整合行銷、PM...等，約收過 40、50 封以上履歷。

從收履歷經驗中，有些人讓我極想和他合作，有些人則一開始就讓我扣分，我來分享三個實際碰到的應徵情境，也讓你評估自己會如何選才：

一、敲我的那一刻，就開始篩選了


我不是很喜歡篩選這個詞，但是職缺有限，不可能每個來應聘的人都接受，於是還是會為每個人打分數 / 分類。

當 A 敲我時，是說：「我有看到你的分享，希望有機會可以合作。」

B 則說：「本人目前是全職 freelancer，是一位 Full-stack web developer，前端主要使用的 vuejs + tailwind，經歷與作品如下...」

我在徵才文中，明確要求提供供履歷與工作狀態。要是你，會為誰加分？

很顯然 B 得分了。因為他主動提供資訊讓我認識。而當我向 A 要作品，他才給作品；要經歷後，才給經歷。

若我和 A 合作，我已經可以預見將來會有繁瑣溝通，而默默往後排 A 的合作順位。因為，他很可能無法單獨面對案主，而需要我費心思居中溝通。

除非他的能力夠強，才可能說服我找他合作。

二、自信


當我問 C 是否具備某能力時，C 回答：「沒有用過...」

D 則回答：「抱歉，目前沒有使用過。不過我想，學習應該不會是很困難的事情。」

要是你，誰會加分？

做為一個工作內容日新月異的工程師，具備能夠學會任何技術的自信，是會讓人想要合作的。

有幾位工程師還特別有自信地說：

「我切版能力非常強，我的網頁從來不跑版，不管 windows 怎麼縮，也完美符合設計師要求。」

「我專精 js 排程控制部分，有設計稿的話，自認可以完整複刻畫面。」

有自信說出這些話的人，我會格外想合作。

三、能力


應徵者在擁有溝通能力、自信下，我會特別認真看他的履歷與作品。

例如找前端工程師時，我會看作品的製作細節，以及是否實現過特別厲害的功能；找設計師則會看擅長風格，以及作品是否有對應的設計概念。

如果非我專長領域，則會找相同專業的朋友協助評估鑑定。

看作品時，也遇過兩種常見類型：

E 的履歷中，列出自己會的技能，是知名大學畢業，但是只有提供少量作品集，有些內容還不完整。

F 的履歷則有豐富作品，並且詳細描述自己負責的工作項目，甚至附上圖片讓人一目了然。

要是你，會偏好哪位？

如果雙方能力都達標的情況下，我會偏好 F，因為他的做事細節讓人放心，我不必多問，透過作品就能清楚他的能力。E 則需要我多費時評估詢問，我沒有那麼多時間來回追問。

缺乏軟實力，就只能實力過硬


看完這三點，你會發現我特別看重軟實力，而硬實力只要達成標準就可以。為什麼？

因為能力很容易可以成長，但是軟實力難以培養。

缺點並不是絕對的應徵標準，如果一個人缺乏溝通能力、自信、積極，就需要用比其他人更多倍的能力來彌補差距。

並且，工作畢竟是團體戰，光有能力是不夠的。相信工作過的人，都深刻了解一個友善同事有多重要。

有些公司沒有選擇你合作，不一定是能力未達標，而是剛好有人比你更適合那麼一點點。畢竟職缺有限，沒應徵上，不代表你不夠好。

良好溝通能力、有自信，用面試官角度嚴格審視、整理履歷與作品，達成上述三點，相信找到工作與合作機會，會變得容易許多。
